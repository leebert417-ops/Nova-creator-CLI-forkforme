## 📦 角色卡打包说明

完成所有创作内容后，需要使用打包程序将文件整合成 SillyTavern 可导入的角色卡 JSON 文件。

### 第一步：编写配置文件

在项目根目录创建一个 YAML 配置文件（例如：`榜一大哥模拟器.yaml`），内容参考以下格式：

```yaml
# 基础信息
name: 榜一大哥模拟器
creator: ""
character_version: ""

# 主要字段映射
fields:
  description: 作品/榜一大哥模拟器/角色设定_主角名.xyaml
  personality: ""
  scenario: ""
  first_mes: 作品/榜一大哥模拟器/开场白.md
  mes_example: ""
  creator_notes: ""
  system_prompt: ""
  post_history_instructions: ""

# 扩展字段
extensions:
  talkativeness: "0.5"
  fav: false
  world: 榜一大哥模拟器
  status_bar: 作品/榜一大哥模拟器/状态栏.html  # 可选

# 角色书配置
character_book:
  name: 榜一大哥模拟器
  entries:
    # [InitVar]初始化条目
    - comment: "[InitVar]初始化"
      content: 作品/榜一大哥模拟器/变量初始化_beta.xyaml
      enabled: false
      position: before_char
      insertion_order: 100
      depth: 4
      role: 0

    # 变量更新规则
    - comment: "变量更新规则"
      content: 作品/榜一大哥模拟器/变量更新规则.xyaml
      enabled: true
      position: at_depth
      insertion_order: 1
      depth: 1
      role: 0

    # 变量处理指令集
    - comment: "变量处理指令集"
      content: 作品/榜一大哥模拟器/变量处理指令集_beta.xyaml
      enabled: true
      position: at_depth
      insertion_order: 2
      depth: 1
      role: 0

    # 背景设定
    - comment: "背景设定"
      content: 作品/榜一大哥模拟器/背景设定.xyaml
      enabled: true
      position: before_char
      insertion_order: 1
      depth: 4
      role: 0

    # 玩家角色
    - comment: "玩家角色_{{user}}"
      content: 作品/榜一大哥模拟器/玩家角色_{{user}}.xyaml
      enabled: true
      position: before_char
      insertion_order: 2
      depth: 4
      role: 0

    # 其他设定条目（可添加多个）
    - comment: "其他设定条目1"
      content: 作品/榜一大哥模拟器/其他设定1.xyaml
      enabled: true
      position: after_char
      insertion_order: 1
      depth: 4
      role: 0
```

**配置说明：**

- `name`: 角色卡名称
- `fields`: 各个字段对应的文件路径，空字符串 `""` 表示该字段为空
- `extensions.status_bar`: 状态栏HTML文件路径（可选）
- `character_book.entries`: 角色书条目列表
  - `comment`: 条目名称
  - `content`: 条目内容对应的文件路径
  - `position`: 插入位置（`before_char`/`after_char`/`at_depth`）
  - `insertion_order`: 插入顺序（数字越小越靠前）
  - `depth`: 深度值（1-4）
  - `role`: 角色类型（0=系统，1=用户，2=AI）

**参考示例：** `config.example.yaml`

### 第二步：运行打包程序

确保已安装 Node.js 和依赖包，然后在项目根目录运行：

```bash
# 首次使用需要安装依赖
npm install

# 运行打包程序
node build-card.js 榜一大哥模拟器.yaml
```

程序会自动读取配置文件，整合所有内容，生成 `榜一大哥模拟器.json` 文件。
